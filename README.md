# 🎮 DoomArcade — аркадный автомат с Doom-like шутером для Space Station 14 (Goob-Station)

Полноценная мини-игра в стиле Doom, встроенная в игровой мир SS14 в виде аркадного автомата. Реализует raycasting-рендер, движение, стрельбу, врагов с ИИ и сохранение рекорда на сервере.

## 🕹️ Геймплей

- **Вид от первого лица** с raycasting-рендером (как оригинальный Doom/Wolfenstein 3D)
- **Цель:** уничтожить всех врагов на карте, набрать как можно больше очков
- За каждого убитого врага — **+100 очков**, за зачистку волны — **+500 очков**
- После уничтожения всех врагов появляется новая волна
- При смерти игрока — экран Game Over; любая клавиша перезапускает игру
- **Лучший результат** сохраняется на сервере в компоненте автомата

### Управление

| Клавиша | Действие |
|---|---|
| `W` / `↑` | Движение вперёд |
| `S` / `↓` | Движение назад |
| `A` | Стрейф влево |
| `D` | Стрейф вправо |
| `←` | Поворот влево |
| `→` | Поворот вправо |
| `↑` (TextCursorUp) | Выстрел |

## ⚙️ Технические детали

### Рендер (`DoomArcadeControl.cs`)

Рендер реализован через `DrawingHandleScreen` (Robust UI) без использования шейдеров или 3D-движка:

- **Raycasting (DDA-алгоритм):** для каждого столбца экрана пускается луч, определяется расстояние до стены и высота вертикальной полосы
- **Depth buffer:** массив глубин используется при отрисовке спрайтов врагов — спрайт не рисуется, если он за стеной
- **Затемнение:** цвет стен и врагов затемняется пропорционально расстоянию (`1 / (1 + dist * k)`)
- **Боковые стены:** грани, поражённые по Y-оси, отрисовываются на 30% темнее
- **Миникарта:** отображается в левом верхнем углу, показывает стены (серые), игрока (зелёный) и живых врагов (красные)
- **HUD:** прицел (зелёный крест), пистолет и вспышка выстрела рисуются поверх сцены

### Игровая логика (`DoomArcadeGame.cs`)

- **Карта:** статичная сетка 16×16, значение клетки определяет тип стены (0 = пусто, 1–4 = разные цвета)
- **Коллизия:** проверяется отдельно по X и Y с отступом `margin = 0.2f`
- **Стрельба:** hitсcan с проверкой угла (`dot > 0.95`) и трассировкой луча (`IsWallBetween`) до ближайшего врага в радиусе 10 единиц
- **ИИ врагов:** при дальности < 8 единиц враг движется к игроку, при дальности < 1.5 — атакует раз в секунду (−5 HP)
- **Перезапуск волны:** когда все враги мертвы, очки суммируются, враги респавнятся

### Сеть (`DoomArcadeMessages.cs`, `DoomArcadeSystem.cs`)

- При закрытии окна BUI отправляет `DoomArcadeScoreMessage` с финальным счётом
- Сервер (`DoomArcadeSystem`) обновляет `HighScore` в компоненте, если новый счёт выше

## 🗺️ Карта

Карта 16×16, жёстко задана в коде. Условные обозначения:

| Значение | Цвет стены |
|---|---|
| `0` | Пустое пространство |
| `1` | Серый (внешние стены) |
| `2` | Тёмно-красный |
| `3` | Тёмно-зелёный |
| `4` | Тёмно-синий |

## 🧩 Прототип энтити (`doom_arcade.yml`)

Автомат наследуется от `ArcadeBase` и использует стандартный спрайт `arcade.rsi`. Дополнительно:

- `PointLight` с красным цветом (`#d94b3d`) для атмосферы
- `ActivatableUI` + `ActivatableUIRequiresPower` — открывается только при наличии питания
- `StationAiWhitelist` — взаимодействие доступно в т.ч. для Station AI

## 🖼️ Текстура врага

Файл: `Resources/Textures/DoomArcade/vulp_enemy.png`

Используется для отрисовки спрайтов врагов. Если текстура не загружается, рендер автоматически переключается на фоллбэк — красный прямоугольник с белыми глазами.

## 🚀 Установка

1. Скопировать файлы в соответствующие директории сборки Goob-Station согласно структуре выше
2. Положить текстуру `vulp_enemy.png` в `Resources/Textures/DoomArcade/`
3. Убедиться, что прототип `doom_arcade.yml` подключён к маппингу или заспавнен через `spawn DoomArcadeMachine`
4. Собрать проект — автомат появится в игре и будет доступен для взаимодействия
